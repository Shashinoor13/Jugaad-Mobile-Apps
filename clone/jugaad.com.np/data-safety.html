Data Safety

Terms for Account Delete Request:

1. Account Deletion Request:

The user can submit a request to delete their account permanently. Once the request is processed, all associated data will be removed, and the account cannot be recovered.

2. Irreversible Action:

Account deletion is an irreversible action. Once the account is deleted, all data associated with the account, including personal information, preferences, and usage history, will be permanently erased and cannot be recovered.

3. Grace Period:

In some cases, a grace period may be provided before the account deletion takes effect. During this period, the user can cancel the deletion request by following the provided instructions. After the grace period expires, the account deletion process will proceed.

4. Third-Party Integrations:

f the user's account is linked to any third-party applications or services, the user may need to separately delete their account or revoke access to these services.

Data Safety for Android Application:

1. Data Encryption:

The Android application utilizes industry-standard encryption techniques to safeguard user data, ensuring that it is stored and transmitted securely.

2. Secure Authentication:

The application employs strong authentication mechanisms to protect user accounts from unauthorized access. This may include password hashing, multi-factor authentication, or biometric authentication, depending on the device capabilities.

3. Data Minimization:

The application follows the principle of data minimization, collecting and storing only the necessary user data required for providing the intended services. Any unnecessary or sensitive data is not collected or stored.

4. Consent and Permissions:

The application obtains user consent and permissions before accessing or sharing any personal data. Users have the ability to review and modify these permissions at any time through the device settings.

5. Data Deletion:

Users have the right to request the deletion of their data. Upon receiving such a request, the application will promptly delete all associated personal data from its servers and databases, as per the applicable data protection laws.

6. Regular Security Audits:

The application undergoes regular security audits and assessments to identify and address any vulnerabilities. This ensures that the app remains up-to-date with the latest security practices and standards.

8. Data Breach Response:

In the event of a data breach, the application has measures in place to detect, respond, and mitigate the impact of such incidents. Users will be promptly notified about any potential risks or actions they should take.

9. Privacy Policy:

The application has a comprehensive and transparent privacy policy that outlines the types of data collected, how it is used, and the security measures in place. Users are encouraged to review and understand this policy.
